## 3. Функціональні вимоги

### 3.1 Маршрути та навігація
Маршрути:
- `/`
- `/login`
- `/register`
- `/admin`
- `/admin/dashboard`
- `/admin/users`
- `/admin/user/:id`
- `/admin/reports`
- `/:pathMatch(.*)*` — NotFoundPage

Налаштовано navigation guard: перевіряє авторизацію користувача перед входом у `/admin`.

### 3.2 Автентифікація
Імітація авторизації:
- Login: email + password, мінімальна перевірка
- Реєстрація: збереження користувача у localStorage

### 3.3 Вкладки Dashboard
- Дві вкладки: Overview, Activity
- Збереження стану через KeepAlive

### 3.4 Взаємодія між компонентами
- Від батьків → дитині: props
- Від дитини → батька: emits
- Механізм v-model у окремому компоненті
- provide/inject (дані про користувача)
- scoped-slot у Reports
