# 4. Стори та керування станом (Pinia)
## 4.1 Ініціалізація Pinia
Використано createPinia() у main.ts
Pinia підключено глобально до застосунку

## 4.2 Використання defineStore
Ствoрено такі стори:

- profileStore — дані профілю користувача
- cartStore — список товарів і кошик
- settingsStore — мова застосунку (locale)

## 4.3 storeToRefs
storeToRefs() використано для коректного реактивного доступу до стану
Застосовано у Profile та Cart сторінках

## 4.4 persisted state
Стан зберігається у localStorage
Використано:
- pick — для збереження лише необхідних полів

- custom key — окремий ключ для кожного стора

Дані профілю, кошика та мови не втрачаються при перезавантаженні сторінки

# 5. Локалізація (Vue I18n)

## 5.1 createI18n
I18n ініціалізовано через createI18n()
Створено два словники:
- ua.json
- en.json

## 5.2 messages
Тексти профілю, товарів, помилок валідації та кнопок локалізовані
Використано $t() у шаблонах компонентів

## 5.3 Кастомний модифікатор
Реалізовано кастомний формат для виводу значень (наприклад, текстів або чисел)

## 5.4 Синхронізація з settings.locale
Поточна мова зберігається у settingsStore.locale
При зміні мови:
- оновлюється I18n locale
- значення зберігається у localStorage

# 6. Валідація форм (VeeValidate)

## 6.1 configure()
Глобально налаштовано повідомлення про помилки

## 6.2 useForm / useField
Використано для форми профілю:
- імʼя
- email
- дата народження

## 6.3 useFieldArray
Реалізовано масив телефонів
Можливість додавання та видалення номерів

## 6.4 validationSchema
Валідація реалізована через Yup
Перевіряється:
- обовʼязковість полів
- формат email
- коректність телефону

## 6.5 setFieldError
Використано для ручного встановлення помилок у формі
